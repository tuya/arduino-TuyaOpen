# 强制门户网站示例

## 功能描述

此示例演示如何使用 DNSServer 库创建强制门户网站（Captive Portal）。强制门户网站常用于 WiFi 网络，将所有 DNS 请求重定向到特定网页，通常用于身份验证或信息展示。

该示例设置一个 WiFi 接入点和 DNS 服务器，将所有域名查询重定向到一个显示"Hello World!"的简单网页。

## 硬件要求

- 具有 WiFi 功能的涂鸦开放平台开发板（ESP32、T2、T3、T5 等）

## 使用说明

1. 修改代码中的 WiFi 凭据：
   ```cpp
   WiFi.softAP("your_ssid","your_passwd");
   ```
   将 `"your_ssid"` 和 `"your_passwd"` 替换为您所需的接入点名称和密码。

2. 将程序上传到开发板。

3. 使用移动设备或计算机连接到该 WiFi 网络。

4. 尝试在浏览器中访问任何网站 - 您将自动重定向到显示"Hello World!"的强制门户页面。

## 功能要点

- **WiFi 接入点**：创建可配置 SSID 和密码的 WiFi AP
- **DNS 服务器**：使用通配符域名匹配（`"*"`）拦截所有 DNS 请求
- **HTTP Web 服务器**：在 80 端口提供简单的 HTML 页面
- **自动重定向**：所有 DNS 查询重定向到 AP 的 IP 地址（8.8.4.4）
- **强制门户检测**：兼容大多数设备的自动强制门户检测功能

## 代码要点

- 使用 `DNSServer` 库捕获所有 DNS 请求
- 设置自定义 AP IP 地址（8.8.4.4 以兼容 Android）
- 实现简单的 HTTP 服务器以响应 HTML 内容
- 在主循环中处理 DNS 请求以实现连续操作
