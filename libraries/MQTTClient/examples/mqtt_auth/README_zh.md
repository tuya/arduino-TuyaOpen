# MQTT 认证示例

## 功能描述

本示例演示如何使用用户名和密码连接到 MQTT 代理服务器。展示了在涂鸦开发板上使用 MQTTClient 库建立安全 MQTT 连接、发布消息和订阅主题的基本设置。

## 硬件要求

- 任意具有 WiFi 功能的涂鸦支持开发板（T2、T3、T5、ESP32 或 XH_WB5E）
- WiFi 网络连接
- 访问 MQTT 代理服务器（示例使用 broker.emqx.io）

## 使用说明

1. 在 Arduino IDE 中打开本示例
2. 修改 WiFi 凭据：
   ```cpp
   const char *ssid = "your_ssid";
   const char *pass = "your_passwd";
   ```
3. 可选修改 MQTT 设置：
   - `mqtt_broker`: MQTT 代理服务器地址
   - `topic`: 发布/订阅的主题
   - `mqtt_username`: MQTT 用户名
   - `mqtt_password`: MQTT 密码
4. 从工具 > 开发板菜单选择您的涂鸦开发板
5. 上传程序到开发板
6. 打开串口监视器，波特率设置为 115200
7. 观察开发板连接到 WiFi 和 MQTT 代理服务器的过程

## 代码功能要点

- **WiFi 连接**：连接到 WiFi 网络并显示 IP 地址
- **MQTT 认证**：演示使用用户名和密码连接到 MQTT 代理服务器
- **客户端配置**：展示如何配置 MQTT 客户端参数：
  - 主机和端口
  - 保持连接间隔
  - 超时设置
  - 客户端 ID
- **发布消息**：向指定主题发布消息
- **订阅主题**：订阅主题以接收消息
- **消息回调**：包含回调函数以处理接收到的消息

## 配置参数

- **Keep-alive（保持连接）**: 15 秒 - 连接心跳间隔
- **Timeout（超时）**: 100 毫秒 - 操作超时时间
- **Port（端口）**: 1883 - 标准 MQTT 端口（未加密）

## 注意事项

- 示例使用公共 MQTT 代理服务器 `broker.emqx.io`，适合测试使用
- 生产应用中，请使用您自己的 MQTT 代理服务器
- 示例展示了认证设置，但所使用的代理服务器可能不需要认证
- 需要实现 `callback` 函数来处理接收到的消息
